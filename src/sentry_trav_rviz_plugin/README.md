# sentry_trav_rviz_plugin

RViz2 (ROS 2 **Jazzy**, Qt5) 插件，用于**手工绘制**可通行性标注区域并读写 `plan_env`
约定的 `.trav.yaml`。标注分三类，与 `sentry_nav::TravType` 一一对应：

| 类型       | 含义                          | 颜色 |
|------------|-------------------------------|------|
| `free`     | 普通可通行（无约束）          | 绿   |
| `obstacle` | 人工补充障碍（双向禁止）      | 红   |
| `oneway`   | 单向可通行（带允许方向+容差） | 蓝 + 方向箭头 |

标注以**世界 / odom 坐标（米）**为单位，约定起点 = 地图原点，故与先验地图直接对齐，
无需额外 TF。

## 组成

- **TraversabilityPanel**（`rviz_common::Panel`）：属性选择、区域列表、读写 yaml、发布预览。
- **TraversabilityTool**（`rviz_common::Tool`）：在 3D 视图里点取多边形顶点。

两者由 pluginlib 各自实例化、互不持有指针，靠一个进程内 Qt 信号单例
`AnnotationBridge` 解耦通信（同一 RViz 进程、同一 GUI 线程）。

## 取点的两种方式

1. **TraversabilityTool（推荐，真正的 RViz Tool）**
   - 左键：把屏幕点投影到地面 `z=0`（用 `rviz_rendering::ViewportProjectionFinder::`
     `getViewportPointProjectionOnXYPlane`，与内置 GoalTool/PoseTool 同一套 Jazzy API），
     追加为当前多边形的一个顶点。
   - 右键 / `Esc`：结束当前多边形。
   - `Backspace`：撤销上一个顶点。
2. **内置 "Publish Point" 工具（兜底/便捷）**
   - Panel 同时订阅 `/clicked_point`（`geometry_msgs/PointStamped`）。处于绘制态时，
     用 RViz 自带的 "Publish Point" 点击也会并入当前多边形。

> 设计选择：优先实现真正的 Tool（地面投影更顺手），并保留 `/clicked_point` 通道做兜底，
> 两条路最终都走 `addPoint()` 同一入口。

## 在 RViz 里加载

构建并 source 工作空间后启动 `rviz2`：

```zsh
colcon build --packages-select sentry_trav_rviz_plugin   # 由整体构建阶段统一执行
source install/setup.zsh
rviz2
```

1. **Panels → Add New Panel → `sentry_trav_rviz_plugin/TraversabilityPanel`**。
2. 工具栏 **`+` → `sentry_trav_rviz_plugin/TraversabilityTool`**（快捷键 `r`）。
3. 加一个 **MarkerArray** 显示，话题设为 `/traversability_annotation_preview`，
   即可实时看到标注（多边形按类型着色 + oneway 方向箭头 + 文字标签）。
4. 把 RViz 的 **Fixed Frame** 设为标注所在坐标系（通常 `map` 或 `odom`）。预览 Marker
   会用当前 Fixed Frame 发布。

## 标注工作流

1. 在 Panel 选 **Type / Direction / Tolerance / Id**（`oneway` 才需要 Direction；
   Tolerance 默认 `90°` = 前向半球锥）。Id 留空会自动命名为 `region_N`。
2. 点 **New Region** → 用 Tool 左键依次点出多边形顶点（≥3 个）。
3. **右键 / Esc / Finish Region** 提交该区域；它出现在区域列表里。
4. 重复 1–3 标注更多区域。**Delete Selected** 可删除列表中选中的区域。
5. **Save .trav.yaml** 导出；**Load .trav.yaml** 可回读继续编辑。

## 导出格式（与 `plan_env` 契约一致）

```yaml
# generated by sentry_trav_rviz_plugin
map:
  resolution: 0.05
  origin: [-3.58, -9.44]
  width: 0            # 0 => 由 regions 包围盒自动推导
  height: 0
default_tolerance_deg: 90
regions:
  - id: step_x
    type: oneway
    direction_deg: -90
    tolerance_deg: 90
    polygon: [[x0, y0], [x1, y1], [x2, y2]]   # >=3 个世界坐标顶点
```

- `direction_deg` / `tolerance_deg` 只对 `oneway` 写出（其余类型由加载侧取默认值）。
- `width` / `height` 写 `0`，加载时由 `TraversabilityLayer` 依据各 region 包围盒自动推导，
  因此 `origin` 仅作元信息记录，不影响 `free/obstacle/oneway` 查询结果。

`oneway` 方向语义：`direction_deg` 为允许行进朝向（世界系 `atan2(dy,dx)`）；
`tolerance_deg` 为半张角，放行条件 `dot(unit(travel), dir) >= cos(tolerance_deg)`。
`90°` → `cos=0` → 允许前向半球（逆向被挡）。
